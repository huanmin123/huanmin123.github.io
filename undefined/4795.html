组件化

什么叫做组件化

什么叫做组件化，为什么要组件化？接下来我就针对这两个问题一一解答，所谓组件化，就是把页面拆分成多个组件，每个组件依赖的 CSS、JS、模板、图片等资源放在一起开发和维护。 因为组件是资源独立的，所以组件在系统内部可复用，组件和组件之间可以嵌套，如果项目比较复杂，可以极大简化代码量，并且对后期的需求变更和维护也更加友好。

最简单的理解就是 :自定义组件其实就是 自定义标签 而这个标签在内部 已经被实现了 一些功能 和css美化 拿来直接就可以使用

就比如html中的<u></u> 这个标签是默认添加下划线 我们也可以通过自定义组件来完成这样的效果

来体验下效果:


        <div id="app">
            <!--使用定义好的全局组件 对hello进行添加下划线  -->
            <countnum text="hello" ></countnum>
        </div>
        <script src="./jquery-3.4.1.js"></script>
        <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
        <script type="text/javascript">
            // 定义全局组件，两个参数：1，组件名称。2，组件参数
            Vue.component("countnum", {
                template: '<span v-html="text"  v-u></span>',
                 props:['text'],
                 directives:{
                     u:{
                         inserted:function(el){
                             $(el).css("text-decoration","underline")
                         }
                     }
                 }
            })
            var app = new Vue({
                el: "#app"
            })
        </script>

以上代码中 directives 我们已经学过了 在基础Vue里 自定义指令

至于其他的 我们下面就慢慢来 一起探讨

组件注册

使用自定义组件之前必须注册。
那么如何注册一个组件呢？ Vue.js 提供了 2 种组件的注册方式，全局注册和局部注册

特别注意 : 组件的名称 如果全部小写的话 countnum 那么直接调用就行了 <countnum></countnum>

如果是 大小写混合 比如 CountNum那么调用的时候使用 <Count-Num></Count-Num>

  • 组件其实也是一个Vue实例，因此它在定义时也会接收：data、methods、生命周期函数等
  • 组件不会与页面的元素绑定，否则就无法复用了，因此没有el属性。 而组件本身就是一个标签
  • 但是组件渲染需要html模板，所以增加了template属性，值就是HTML模板
  • 全局组件定义完毕，任何vue实例都可以直接在HTML中通过组件名称来使用组件了。
  • data的定义方式比较特殊，必须是一个函数。

全局注册

全局注册 是能在当前 页面下所有Vue 实例范围内都能使用的

在 vue.js 中我们可以使用 Vue.component(tagName, options) 进行全局注册，例如

                    }
                }

                el: "#app"

局部注册

一旦全局注册，就意味着即便以后你不再使用这个组件，它依然会随着Vue的加载而加载。

因此，对于一些并不频繁使用的组件，我们会采用局部注册。 而局部注册 你的组件只能 作用在当前Vue实例的范围内

我们先在外部自定义主键参数对象，结构与全局组件参数一致 唯一不同的是 不是将内容给Vue了而 是给当前Vue的实例了

会随着这个Vue实例结束而消失

        <div id="app">
            <!--使用定义好的局部组件  -->
            <countnum></countnum>  <br />
            <countnum></countnum><br />
        </div>
        <script src="./jquery-3.4.1.js"></script>
        <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
        <script type="text/javascript">
            var countnumObj = {
                template: '<button v-on:click="count++">你点了我 {{ count }} 次，我记住了.</button>',
                data() {
                    return {
                        count: 0
                    }
                }
            };
            var app = new Vue({
                el: "#app",
                components: {
                    // 主键名称  :主键参数对象
                    countnum: countnumObj // 将定义的对象注册为组件
                }
            })
        </script>

组件通信

对于vue来说，组件之间的消息传递是非常重要的，

什么是父组件 什么是子组件

父组件: 就是当前.html文件或者.vue文件 这么理解就行

子组件 : 就是我们在 .html 或着.vue文件内 自己定义 一个内部组件

主要用到 props:[]

详细一点解释就是：

一个组件可以直接在模板里面使用data里面的数据

但是子组件不能 使用父组件中data 里面的数据。

如果子组件想要引用父组件的data数据，那么就在子组件prop里面声明一个变量（比如a），这个变量就可以引用父元素的数据。 我们在子组件直接操作这个变量就行 和data使用一样

包括通过父组件 将 属性值传递给 子组件

父组件 将 值传递给 子组件(常用)

因为组件是自定义的 所以 属性也是自定义 那么如何在 在组件代码内获取组件属性值呢? 通过props 获取

子向父的通信

因为无法在子组件中 修改num的值 会报错的 所以我们需要在 vue实例中创建方法 让他自己修改

而我们子组件 只负责调用 也可以说是启动而已 我们先看代码

        <div id="app">
            <!-- 父组件 -->
            <h2>num: {{num}}</h2>
            <!--子组件 -->
            <counter @jia="increment()" @jian="decrement()"></counter>
        </div>

@jia 这个是啥意思 和 v-bind 一样 是 v-on简写 而事件是自己定义的事件没啥用就是个名称

用于绑定Vue中的方法然后在组件内部 通过this.$emit(“事件名称”) 来触发自定义事件 启动 Vue实例 中的方法

而我们将这段代码 放入组件内 方法中在template 里就可以通过 事件来调用组件方法 从而调用 Vue实例中的方法 从而改变 父组件的 值

        <script src="./jquery-3.4.1.js"></script>
        <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
        <script type="text/javascript">
            Vue.component("counter", { // 子组件，定义了两个按钮，点击数字num会加或减
                template: '<div>'+
                          '<button @click="plus">加</button> ' +
                          ' <button @click="reduce">减</button>'+
                          '</div>',
                methods: {
                    plus() {
                        this.$emit("jia"); //触发加事件
                    },
                    reduce() {
                        this.$emit("jian"); //触发减事件
                    }
                }
            })
            var app = new Vue({
                el: "#app",
                data: {
                    num: 0
                },
                methods: {
                    increment: function() {
                        this.num++;
                    },
                    decrement: function() {
                        this.num--;
                    },
                }

            })
        </script>

vue-router的使用

新建vue-router对象，并且指定路由规则：

// 创建VueRouter对象
var router = new VueRouter({
    routes:[ // 编写多个路由规则
        {
            path:"/login", // 请求路径
            component:loginForm // 组件名称
        },
        {
            path:"/register",
            component:registerForm
        },
    ]
})
  • 创建VueRouter对象，并指定路由参数
  • routes：路由规则的数组，可以指定多个对象，每个对象是一条路由规则，包含以下属性：
    • path：路由的路径
    • component：组件名称

注意: 路由规则中: path: ‘/‘, 是当前页面直接显示内容 /xxx就是需要通过

<router-link to="/xxx">router</router-link>跳到此路径 才能显示 内容

和a标签使用一样 设置样式 #app a{ } 就是设置 router-link

而router-view 就是一个占位符 需要通过标签来包起来 然后才能移动

vue-router 要在 放在 Vue对象实例之前 否则无效 以及报错

在父组件中引入router对象：

var vm = new Vue({
    el:"#app",
    components:{// 引用登录和注册组件
        loginForm,
        registerForm
    },
    router // 引用上面定义的router对象
})

页面跳转控制：

<div id="app">
    <!--router-link来指定跳转的路径-->
    <span><router-link to="/login">登录</router-link></span>
    <span><router-link to="/register">注册</router-link></span>
    <hr/>
    <div>
        <!--vue-router的锚点-->
        <router-view></router-view>
    </div>
</div>

  • 通过<router-view>来指定一个锚点，当路由的路径匹配时，vue-router会自动把对应组件放到锚点位置进行渲染

  • 通过<router-link>指定一个跳转链接，当点击时，会触发vue-router的路由功能，路径中的hash值会随之改变

webpack(打包)

介绍

Webpack 是一个前端资源的打包工具，它可以将js、image、css等资源当成一个模块进行打包。

中文官方网站：https://www.webpackjs.com/

为什么需要打包？

  • 将许多碎小文件打包成一个整体，减少单页面内的衍生请求次数，提高网站效率。
  • 将ES6的高级语法进行转换编译，以兼容老版本的浏览器。
  • 将代码打包的同时进行混淆，提高代码的安全性。

在简单来理解就是 将多个文件 整合成一个文件 因为每次加载网页的时候 每个文件都是一次请求 而将多个文件整合后就能减少请求 加快网页速度

打包的四个核心概念

  • 入口(entry)

    webpack打包的启点，可以有一个或多个，一般是js文件。webpack会从启点文件开始，寻找启点直接或间接依赖的其它所有的依赖，包括JS、CSS、图片资源等，作为将来打包的原始数据

  • 输出(output)

    出口一般包含两个属性：path和filename。用来告诉webpack打包的目标文件夹，以及文件的名称。目的地也可以有多个。

  • 加载器（loader）

    webpack本身只识别Js文件，如果要加载非JS文件，必须指定一些额外的加载器（loader），例如css-loader。然后将这些文件转为webpack能处理的有效模块，最后利用webpack的打包能力去处理。

  • 插件(plugins)

    插件可以扩展webpack的功能，让webpack不仅仅是完成打包，甚至各种更复杂的功能，或者是对打包功能进行优化、压缩，提高效率。

一个页面多个 router-view

先来说下规则 router-view 是依据 router 配置里的 path 来访问的

也就是 多个router-view 要想 同时显示 那么 就需要path 一样

否则不同的path 的 router-view 就会消失

因为 每次一 路由 都相当于是一次新的请求 原来的 路由数据没有了 只是这请求 是在vue内部 感觉不到而已

如果还不理解 那么看图:

component 中有 default 是默认加载到 没有设置name的 router-view

一个页面就是一个网站 –毕业案

而且影响用户体验

我们下面这个案例的 公共路径路径使用的是 空 也就是相对路径 打包后直接就能点开运行
