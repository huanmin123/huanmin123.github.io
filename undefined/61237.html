java常用的泛型集合(map List...)

集合中常用的接口

世间上本来没有集合,(只有数组参考C语言)但有人想要,所以有了集合

有人想有可以自动扩展的数组,所以有了List

有的人想有没有重复的数组,所以有了set

有人想有自动排序的组数,所以有了TreeSet,TreeList,Tree…

而几乎有有的集合都是基于数组来实现的.

因为集合是对数组做的封装,所以,数组永远比任何一个集合要快

Collection 是所有单列集合的顶级接口

储存不唯一 , 无序的对象

Set 接口 (单列)

1.储存唯一 ,无序 的对象

类:TreeSet(排序),HashSet(无序 效率),LinkedHashSet(有序)

List 接口(单列)

储存不唯一 , 有序的对象

类:Vector(同步 安全),ArrayList(查询,修改 效率),LinkedList(增删 效率)

Map 接口(双列)

储存成对 的 键-值 键唯一 值不唯一

类:HashMap(效率),Hashtable(安全),LinkedHashMap(有序)

这些类的使用都差不多

toString

用于获取整个集合中的值 一般用于检查集合中的内容情况

Map<String ,Object> map=new HashMap<>();

List 接口

List是接口 里面 只是定义个方法并没有实现 所以需要子类 来实现

ArrayList类 来实现它 还有一个实现类是LinkedList 但是使用最多的还是ArrayList

定义方式:

List list=new ArrayList();  //不指定值的类型  默认为Object类型
List<String> list1=new ArrayList<>(); //指定值的类型
List<javaBean> list2=new ArrayList<>(); //指定值是对象类型

增加方法 返回boolean

使用map集合映射对象成员属性

日常中我们 向对象里赋值的时候 都是一个一个的赋值的 这样很累

但是java中有一个方法可以解决这个问题 就是 BeanUtils.populate()

但是要求对象属性名称必须和map中的key一致而且还要有set和方法

必须导入import org.apache.commons.beanutils.BeanUtils;
如果没有可以去下载 直接搜索 maven beanutis
如果使用 maven 的话 我这里提供一个坐标赋值上去就行了

commons-beanutils
commons-beanutils
1.9.3

 Map<String,Object> map=new HashMap<>();
 map.put("name","abc");
 map.put("age","22");
 map.put("qq","13532425");
 User user=new User();
 try {
//将map中的值按照key来映射到对象中   
对象中没有匹配到key的中全部为初始状态
     BeanUtils.populate(user,map);
 } catch (IllegalAccessException | InvocationTargetException e) {
     e.printStackTrace();
 }
 System.out.println(user.toString());

在web中前端传递到后端的参数 可以使用map将值映射到对应的对象中

        Map<String,String[]> map= request.getParameterMap();

            e.printStackTrace();
        }
        System.out.println(bean.toString());

Set接口

set接口没有获取值的方法 只有添加删除..方法 如果想要获取值只能通过遍历

使用set最常用的几种情况

​ 将数组或者List值存入然后遍历

      Integer[] a={1,2,34};
        Set<Integer> set= new HashSet<>(Arrays.asList(a));
        for (int i:set){
            System.out.println(i);
        }
   List<Integer> list = new ArrayList<>();
        list.add(1);
        list.add(2);
        list.add(3);
        Set<Integer> set = new HashSet<>(list);
        for (int i : set) {
            System.out.println(i);
        }

将Map中的key 或者value存入对应的集合中

 Map<String,Object> map=new HashMap<>();
        map.put("a",1);
        map.put("b",2);
        map.put("c",3);
        //将map中的key 存入set中
        Set<String> set= map.keySet();
        for (String i:set){
            System.out.println(i);
        }
        Map<String,Object> map=new HashMap<>();
        map.put("a",1);
        map.put("b",2);
        map.put("c",3);
        //将map中的value 存入 Collection中
        Collection<Object> coll=  map.values();
        for (Object i:coll){
            System.out.println(i);
        }

有一个好玩的工具类的方法,将集合中的元素随机打乱 注意必须是单列集合

    List<String> list=Arrays.asList("a","b","c","d","e");
        Collections.shuffle(list);
        System.out.println(list);
//[e, b, c, a, d]

集合扩容问题

只有底层用到动态数组结构的 才需要扩容

关于扩容的问题 在程序中 比如HashMap 他的默认容积是16,加载因子的默认是0.75

16*0.75=12 也就是你的HashMap的长度大于等于12了 就会进行扩容在原有的基础上乘2 在做扩容的时候会生成一个新的数组，原来的所有数据需要重新计算哈希码值重新分配到新的数组，所以扩容的操作非常消耗性能.

就需要给他指定一个 满足业务的容积

创建一个空的HahMap设置初始容积为 100……

Map map=new HashMap(100);

List list=new ArrayList(100);

Set list=new HashSet(100);
