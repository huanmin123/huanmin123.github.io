创建Vue实例

每个 Vue 应用都是通过用 Vue 函数创建一个新的 Vue 实例开始的：

var vm = new Vue({

})

在构造函数中传入一个对象，并且在对象中声明各种Vue需要的数据和方法，包括：

- el

- data

- methods

- computed

- beforeCreate()

- created ()

- watch

等等

还有就是可以通过Vue实例对象调用 对象内的数据 下面会有演示

el 绑定模板

每个Vue实例都需要关联一段Html模板，Vue会基于此模板进行视图渲染。

我们可以通过el属性来指定。

例如一段html模板：

<div id="app">

</div>

然后创建Vue实例，关联这个div

var vm = new Vue({

    el:"#app"

})

这样，Vue就可以基于id为app的div元素作为模板进行渲染了。在这个div范围以外的部分是无法使用vue特性的。

data 数据响应

当Vue实例被创建时，它会尝试获取在data中定义的所有属性，用于视图的渲染，并且监视data中的属性变化，当data发生改变，所有相关的视图都将重新渲染，**这就是“响应式“系统。**

computed 计算

computed可以完成各种复杂的逻辑，包括运算、函数调用等，只要最终返回一个结果就可以。下面是一个简易的计算器，来实现乘法的操作

computed内方法，在调用时 不加 () 而且方法不能有参数声明 必须有返回值

在做项目过程中，有时会出现同一个需求，在methods属性中写方法可以实现需求，在computed属性中写方法也可以实现需求。那么，既然在methods属性中可以实现我想要的需求，为什么还会有computed计算属性呢？接下来，我们一起看看两者之间有什么区别。

methods中的方法：每调用一次，就触发一次。

computed 具有缓存功能，在系统刚运行的时候调用一次。只有只有当计算结果发生变化才会被调用

总结

也就是computed 更适合进行数据的处理 因为有缓存 所以不用每次都去重新加载方法 只有结果不一样的时候才会 重新加载方法 一般结果的变化都是依赖于传递的参数 这个参数只能是 data 里的变量

也就是computed适合做数据处理 而 methods适合做逻辑处理

注意 :在系统刚运行的时候调用一次 将计算结果保存到缓存里

但是如果调用 失败 或者 其他逻辑导致不满住条件 导致没有返回结果

那么将不会有 计算结果发生变化 自动调用 因为 都没有缓存成功 怎么对比

watch 响应数据

watch的用法大致有三种。下面代码是watch的一种简单的用法：

侦听属性必须是data中的已存在的数据。

第一种 (常用)

        <div id="app">
            <input type="text" v-model="cityName" /><br />
                新值:<p id="newName"></p>
                原来的值<p id="oldName"></p>    
        </div>
        <script src="./jquery-3.4.1.js"></script>
        <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
        <script type="text/javascript">
            // 生成一个Vue实例
            var app = new Vue({
                // el,即element。要渲染的的页面元素
                el: '#app',
                data: {
                    cityName: 'shanghai'
                },
                watch: {
                    cityName(newName, oldName) {
                        $("#newName").text(newName)
                        $("#oldName").text(oldName)
                    }
                }
            })
        </script>

cityName :监听data中的变量 ..

newName: 新值

oldName:原来的值

第一种有一个缺点 就是当值第一次绑定的时候 不会执行监听函数， 只有当值改变的时候 才会执行

第二种 就解决了第一种的问题

如果我们想在第一次绑定的时候执行此监听函数 则需要 设置
immediate 为 true

                watch: {
                    cityName: {
                        handler(newName, oldName) {
                            $("#newName").text(newName)
                            $("#oldName").text(oldName)
                        },
                        immediate: true
                    }
                }

页面第一次加载就会执行watch

前两种在普通变量的时候 没什么问题但是如使用了 对象之后就发现 监听不了 数组不影响

第三种 解决上面的问题 也就是使用对象也能监听 (深度监听)

注 数组的变化不需要深度监听

        <div id="app">
            <input type="text" v-model="cityName.name"  /><br />
            新值:<p id="newName"></p>
            原来的值<p id="oldName"></p>
        </div>
        <script src="./jquery-3.4.1.js"></script>
        <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
        <script type="text/javascript">
            // 生成一个Vue实例
            var app = new Vue({
                // el,即element。要渲染的的页面元素
                el: '#app',
                data: {
                    cityName: {name:"hu",age:"22"}

                },
                watch: {
                    'cityName.name': {
                        handler(newName, oldName) {
                            $("#newName").text(newName)
                            $("#oldName").text(oldName)
                        },
                        immediate: true,
                        deep: true
                    }
                }
            })

注意不能直接检测 cityName 因为他是对象 你必须检测到 指定对象内 的变量才行

所以使用 ‘cityName.name’ 用双引号括起来

以上是在Vue构造里面写的我们还可以在构造外面调用

Vue指令

什么是指令？

指令 (Directives) 是带有 v- 前缀的特殊

特性。指令特性的预期值是：单个 JavaScript 表达式。指令的职责是，当表达式的值改变时，将其产生的连带影响，响应式地作用于 DOM。

常用的指令有

  • v-html
  • v-text
  • v-bind
  • v-model
  • v-for
  • v-show
  • v-if v-else v-else-if 相当于 java if else if else if ….
  • v-on

v-model 双向绑定

我们在上面估计已经接触了v-model 但是还不了解 到底是干嘛的

v-text和v-html 和{{ }} 可以看做是单向绑定，数据影响了视图渲染，但是反过来就不行。接下来学习的v-model是双向绑定，视图（View）和模型（Model）之间会互相影响。

既然是双向绑定，一定是在视图中可以修改数据，这样就限定了视图的元素类型。目前v-model的可使用元素有：

  • input (单行输入框) 比如test password
  • textarea
  • select
  • checkbox
  • radio
  • components（Vue中的自定义组件）

基本上除了最后一项，其它都是表单的输入项。

双向绑定 也就是 我在 页面上修改 表单输入元素内的值 而Vue内的 对应的值也相应改变 反过来 Vue 内的值发生变化 页面上的内容也会发生变化

自定义指令

指令只一种可以附加到DOM元素的命令 它们通常以v-作为前缀, 以方便Vue知道你在使用一种特殊的标记, 从而确保语法的一致性

一个指令定义对象可以提供如下几个钩子函数 (均为可选)：

bind：页面Vue创建 前调用 在这里可以进行一次性的初始化设置。

inserted：调用时 父节点存在 此时Vue创建完成

同点： dom插入都会调用，bind在inserted之前

不同点：
bind 运行时父节点为不存在

钩子函数参数

指令钩子函数会被传入以下参数：

除了 el 之外，其它参数都应该是只读的，切勿进行修改。如果需要在钩子之间共享数据，建议通过元素的 data 来进行。

el：指令所绑定的元素，可以用来直接操作 DOM 。

binding：一个对象，包含以下属性：

  • name：指令名，不包括 v- 前缀。

  • value：指令的绑定值，

    • 例如：v-my=“1 + 1” 中，绑定值为 2。
    • 例如：v-my=“ ‘aa’ ” 绑定值为aa 注意字符串必须使用单引号括起来 否则就会报错
    • 例如 : v-my=“ str ” 获取data中 str变量 的值 如果不存在就报错
    • 例如 : v-my=”{name:’hu’,age:’21’}” 通过 binding.value.age 获取对象中age的值
    • 列如: v-my=”[‘name’,’pass’]” 数组中是字符串使用单引号括起来否则就会报错 通过 binding.value[0] 获取数组第一位

  • oldValue：指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。

  • expression：字符串形式的指令表达式。例如 v-my-directive=“1 + 1” 中，表达式为 “1 + 1”。

  • arg：传给指令的参数，可选。例如 v-my-directive:foo 中，参数为 “foo”。

  • modifiers：一个包含修饰符的对象。例如：v-my-directive:hello.foo.bar 中，修饰符对象为 { foo: true, bar: true }。

    这些参数 常用的就value 稍微能用到的也就 arg 和 modifiers

                    //删除 购物车里 的 指定商品    其实就是删除 数组中的某一个元素
                    handleRemove: function(index) {    
                        //将选中取消
                        $("[name=\"che\"]")[index+1].checked=false
                        this.cehc()  //从新加载
                        //删除元素
                        this.list.splice(index, 1);
                    },
                    //全部选中  和全部不选中
                    checkAll: function() {
                        var a = $("[name=\"che\"]")
                        if (this.checkpd) {
                            for (var i = 0; i < a.length; i++) {
                                a[i].checked = true
                            }
                            this.checkpd = false;
                            return;
                        }
                        for (var i = 0; i < a.length; i++) {
                            a[i].checked = false;
                        }
                        this.checkpd = true;

                    },
                    //如果 全部都被选中了 那么全选按钮  处于选中状态  
                    //如果 有一个没被选中那么  那么全选按钮  处于未被选中状态    
                    checkisAll: function() {

                        var a = $("[name=\"che\"]")
                        for (var i = 1; i < a.length; i++) {
                            if (a[i].checked == false) {
                                a[0].checked = false
                                return;
                            }
                        }
                        a[0].checked = true
                    },
                    // 记录选中的 下标
                    cehc: function() {
                        var a = $("[name=\"che\"]")
                        this.check = []
                        for (var i = 1; i < a.length; i++) {
                            if (a[i].checked == true) {
                                this.check.push(i)
                            }
                        }
                    }

                },
                computed: {
                    //通过遍历check  从而获取 list中 被选中的商品价格 的总和
                    totalPrice: function() {
                        var num = 0;
                        if (this.check.length > 0) {
                            for (var i = 0; i < this.check.length; i++) {
                                num += this.list[this.check[i] - 1].price * this.list[this.check[i] - 1].count;
                            }
                        }
                        return num;
                    },
                }
            })
        </script>
