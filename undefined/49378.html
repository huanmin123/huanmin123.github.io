java-SpringMVC常用注解

@RequestMapping(访问方式)

请求路径path

通过请求路径可以运行指定方法

1.写在方法上 代表通过此path就可以访问到此方法

    @RequestMapping(path = "/show")
    public String show() {
        System.out.println("hello");
        return "hello";
    }

 <a href="/show">入门Mvc</a>

2.写在类上,代表此类下的所有方法在调用的时候都必须加上类path

@Controller
@RequestMapping(path = "/da1")
public class HeController {

    //请求方式 必须是get 请求  才能请求成功
    @RequestMapping(path = "/show")
    public String show() {
        System.out.println("hello");
        return "hello";
    }

必须包含指定参数

当设置了params 这个属性 就必须传入指定的属性或者属性值 否则 就会报错


参数必须是name名称的参数 值任意
参数必须是age而且值必须是12

    @RequestMapping(path = "/show1", params = {"name", "age=12"})
    public String show1() {
        System.out.println("hello");
        return "hello";
    }

    //    请求头 中必须包含指定的请求头 才能请求成功

    public String show2() {

占位符{?}

类似于url参数,使用逗号隔开

语法:,@RequestMapping(path = “/hell/{id},{id1}”)

获取占位符的值使用@PathVariable(“”)

    @RequestMapping(path = "/hell/{id}")
    @ResponseBody
    public void sayHello1(@PathVariable("id") String a )  {
        System.out.println("我是第一个");
        System.out.println(a);

    }

    @RequestMapping(path = "/hell/{id},{id1}")
    @ResponseBody
    public void sayHello2(@PathVariable("id") String a , @PathVariable("id1") String a1)  {
        System.out.println("我是第二个");
        System.out.println(a);
        System.out.println(a1);
    }

解决@ResponseBody中文乱码

将以下代码添加到MVC注解的前面

    <!--注解适配器 解决@ResponseBody中文乱码-->
    <bean class="org.springframework.web.servlet.mvc.method.annotation.RequestMappingHandlerAdapter">
        <property name="messageConverters">
            <list>
                <ref bean="stringHttpMessageConverter" />
            </list>
        </property>
    </bean>
    <bean id="stringHttpMessageConverter"
          class="org.springframework.http.converter.StringHttpMessageConverter">
        <property name="supportedMediaTypes">
            <list>
                <value>text/html;charset=UTF-8</value>
            </list>
        </property>
    </bean>

@RequestParam(url参数装配)

当请求的参数名称和 方法的参数名不一致的时候 可以使用@RequestMapping来和请求的url参数名称进行配对

比如我的请求<a href="/da1/show3?name1=hu&age=22">参数请求</a>

不使用注解的方式

    @RequestMapping(value = "/show3")
    @ResponseBody
    public void show2(String name, String age) {
        System.out.println("name:" + name + "_" + "age:" + age);
    }

结果:name:null_age:22

使用注解将url 参数 name1 装配到方法 name里

 @RequestMapping(value = "/show3")
@ResponseBody
    public void show2(@RequestParam("name1")String name, String age) {
        System.out.println("name:" + name + "_" + "age:" + age);
    }

结果:name:hu_age:22

@RequestParam还有两个属性没什么用

第一个是required 默认值为true 必须有此Param名称 否则报错400

为false的时候如果没有找对应的Param名称到返回null

    @RequestMapping(value = "/parameter")
    @ResponseBody
    public void  parameter(@RequestParam(value = "name1",required = false) String name)  {
        System.out.println(name);
    }

<a href="/parameter">提交</a>

第二个值是defaultValue默认值

如果没有找到对应的Param名称 或者 没有赋值那么就使用默认值

   @RequestMapping(value = "/parameter")
    @ResponseBody
    public void  parameter(@RequestParam(value = "name1",defaultValue = "abc") String name)  {
        System.out.println(name);
    }
<a href="/parameter">提交</a>

他们的区别是:

required 没有找到对应的名称 那么就返回null

defaultValue 不管有没有对应的Param都会给一个初始值 如果有对应的name没有赋值的话还是使用的初始值 如果赋值了的话就会将初始值覆盖

他俩使用一个就行了 如果都使用的话 只会执行defaultValue

@ModelAttribute 数据初始化

要的使用 方式 就是 如果前端没有传入某些值 那么就使用 原来的值

比如修改 : 我就修改2个字段 其余的字段 还是使用原来的值

在比如检测 传入的值是否合法

@ModelAttribute代码快 ,类似于构造方法每次new对象都会调用但是区别是

@ModelAttribute代码块 每次使用方法都会调用全部的@ModelAttribute代码块

@SessionAttributes(会话)

@ModelAttribute注解作用在方法上或者方法的参数上，表示将被注解的方法的返回值或者是被注解的参数作为Model的属性加入到Model中，然后Spring框架自会将这个Model传递给ViewResolver。Model的生命周期只有一个http请求的处理过程，请求处理完后，Model就销毁了。。

@SessionAttributes能让数据在修饰的类下所有方法都能访问到

可通过Model 或者HttpSession 来访问

package com.htsa.cn.controller;


import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;
import org.springframework.web.bind.annotation.SessionAttributes;
import org.springframework.web.bind.support.SessionStatus;

import javax.servlet.http.HttpSession;

@Controller
@SessionAttributes(value = {"name","age"})
public class HeController {

    @RequestMapping("/session1")
    @ResponseBody
    public void sessions1(Model model) {
        model.addAttribute("name", "hy");
        model.addAttribute("age", 22);
    }

    @RequestMapping("session2")
    @ResponseBody
    public void sessions2(Model model, SessionStatus status,HttpSession session) {
        String name = (String) model.getAttribute("name");
        Integer age = (Integer) model.getAttribute("age");
        String name1 = (String) session.getAttribute("name");
        Integer age1 = (Integer) session.getAttribute("age");
        System.out.println("model:"+name);
        System.out.println("model:"+age);
        System.out.println("sess:"+name1);
        System.out.println("sess:"+age1);
        status.setComplete();//清空 @SessionAttributes 内数据

    }

    @RequestMapping("session3")
    @ResponseBody
    public void sessions3(Model model, SessionStatus status,HttpSession session) {
        String name = (String) model.getAttribute("name");
        Integer age = (Integer) model.getAttribute("age");
        String name1 = (String) session.getAttribute("name");
        Integer age1 = (Integer) session.getAttribute("age");
        System.out.println("model:"+name);
        System.out.println("model:"+age);
        System.out.println("sess:"+name1);
        System.out.println("sess:"+age1);

    }

}
