java-Spring框架IOC

在了解IOC之前推荐先看一篇文件java类工厂模式(反射)能让你更好的了解IOC

什么是Ioc

首先想说说IoC（Inversion of Control，控制倒转）。这是spring的核心，贯穿始终。所谓IoC，对于spring框架来说，就是由spring来负责控制对象的生命周期和对象间的关系。这是什么意思呢，举个简单的例子，我们是如何找女朋友的？常见的情况是，我们到处去看哪里有长得漂亮身材又好的mm，然后打听她们的兴趣爱好、qq号、电话号、ip号、iq号………，想办法认识她们，投其所好送其所要，然后嘿嘿……这个过程是复杂深奥的，我们必须自己设计和面对每个环节。传统的程序开发也是如此，在一个对象中，如果要使用另外的对象，就必须得到它（自己new一个，或者从JNDI中查询一个），使用完之后还要将对象销毁（比如Connection等），对象始终会和其他的接口或类藕合起来。

那么IoC是如何做的呢？有点像通过婚介找女朋友，在我和女朋友之间引入了一个第三者：婚姻介绍所。婚介管理了很多男男女女的资料，我可以向婚介提出一个列表，告诉它我想找个什么样的女朋友，比如长得像李嘉欣，身材像林熙雷，唱歌像周杰伦，速度像卡洛斯，技术像齐达内之类的，然后婚介就会按照我们的要求，提供一个mm，我们只需要去和她谈恋爱、结婚就行了。简单明了，如果婚介给我们的人选不符合要求，我们就会抛出异常。整个过程不再由我自己控制，而是有婚介这样一个类似容器的机构来控制。Spring所倡导的开发方式就是如此，所有的类都会在spring容器中登记，告诉spring你是个什么东西，你需要什么东西，然后spring会在系统运行到适当的时候，把你要的东西主动给你，所有的类的创建、销毁都由spring来控制，也就是说控制对象生存周期的不再是引用它的对象，而是spring。对于某个具体的对象而言，以前是它控制其他对象，现在是所有对象都被spring控制，所以这叫控制反转。

Spring中提供了两种IoC容器：

  • BeanFactory

  • ApplicationContext(常用)

    容器的作用就是将配置文件内的bean加载到内存中的,所以一般定义全局变量 加载一次就行了

他们的区别:

使用ApplicationContext在第一次加载ApplicationContext.xml配置文件 的时候就会将所有对象加载到内存中 (常用)

第一次调用对象会相对应慢些 但是后面调用时候就无需在创建了直接使用就行了

使用BeanFactory调用ApplicationContext.xml文件不会创建对象 只会在调用指定对象的时候 ,才加载对应的对象

就是每次用户调用对象的时候的速度都是一样的 .

第三步从spring获取对象


public class test {
    ApplicationContext context=new ClassPathXmlApplicationContext("ApplicationContext.xml");
    Resource resource=new ClassPathResource("ApplicationContext.xml");
    BeanFactory factory=new XmlBeanFactory(resource);
    @Test
    public void show1(){
        IAccountDao accountDaoJDBCImpl = (IAccountDao) context.getBean("AccountDaoJDBCImpl");
        accountDaoJDBCImpl.saveAccount();
    }

    @Test
    public void show2(){   
        IAccountDao accountDaoJDBCImpl = (IAccountDao)factory.getBean("AccountDaoJDBCImpl");
        accountDaoJDBCImpl.saveAccount();
    }
}

两种容器的使用方法都写出来了

getBean(“id”) id=配置文件中 <bean id

因为getBean返回出来的对象是Object类型 所以需要转类型 (接口或实现类)

Spring创建Bean的三种方式

  1. 第一种使用默认的无参构造也就是我们上面案例使用的方式(常用)

    `注意必须有无参构造,如果你定义了带参构造那么必须显示定义无参构造`

  2. 第二种使用静态工厂的方式创建对象

    是我们自己new对象 只是调用的时候交给Spring

    创建一个类 在类中定义需要new对象的静态方法

    public class StaticFactory {

    public static AccountDaoJDBCImpl  createAccountDaoJDBCImpl(){
        return  new AccountDaoJDBCImpl();
    }
    public static AccountDaoMybatisImpl createAccountDaoMybatisImpl(){
        return  new AccountDaoMybatisImpl();
    }
}

    在配置文件中需要指定属性factory-method=”方法名”来调用方法

    <bean id="factoryAccountDaoMybatisImpl" class="com.htsa.www.Bens.factory.StaticFactory" factory-method="createAccountDaoMybatisImpl" />
        @Test
    public void show3(){
        ApplicationContext context=new ClassPathXmlApplicationContext("ApplicationContext.xml");

        IAccountDao accountDaoJDBCImpl = (IAccountDao) context.getBean("factoryAccountDaoMybatisImpl");
        accountDaoJDBCImpl.saveAccount();
    }

  3. 使用实例工厂的方式创建对象

    这种方式稍微有点绕 和静态工厂不同的是 方法不在是静态的了

    public class livingFactory {

        return  new AccountDaoJDBCImpl();
    }
    public  AccountDaoMybatisImpl createAccountDaoMybatisImpl(){
        return  new AccountDaoMybatisImpl();
    }
}

    bean 引入bean 使用 factory-bean=”bean id”

        <bean id="livingFactory" class="com.htsa.www.Bens.factory.livingFactory"/>
    <bean id="factoryAccountDaoJDBCImpl" factory-bean="livingFactory" factory-method="createAccountDaoJDBCImpl"/>

Spring-bean依赖注入

①使用构造函数的方式来依赖注入

利用 <constructor-arg name=”属性名称” value=”值”/> 来给构造赋值

创建一个类

public class Account_structure {

    private  String name;
    private  Integer age;
    private  double  money;


    public Account_structure(String name, Integer age, double money) {
        this.name = name;
        this.age = age;
        this.money = money;
    }

    @Override
    public String toString() {
        return "Account_structure{" +
                "name='" + name + '\'' +
                ", age=" + age +
                ", money=" + money +
                '}';
    }
}

在Spring配置文件中添加

   <bean id="Account_structure" class="com.htsa.www.Bens.impl.Account_structure">
        <constructor-arg name="name" value="胡"/>
        <constructor-arg name="age" value="22"/>
        <constructor-arg name="money" value="1000.22"/>
    </bean>
    @Test
    public void show5(){
        ApplicationContext context=new ClassPathXmlApplicationContext("ApplicationContext.xml");
        Account_structure account_structure = (Account_structure) context.getBean("Account_structure");
        System.out.println(account_structure);
    }

Account_structure{name=’胡’, age=22, money=1000.22}

③p命名空间注入

p命名空间不能给数组集合等容器初始化

使用p前要在Spring配置文件中引入P的命名空间

xmlns:p=http://www.springframework.org/schema/p

p命名空间的特点:使用属性来配置Bean的属性 而不是使用子元素的形式配置Bean的属性,从而简化了配置代码

必须实体类中 有set方法 才能进行 属性注入

(基本类型 ,字符串 ) p:属性名=”属性值” ;

(对象) p:属性名-ref=”bean id”;

Account_Set类和Account_Ref类在上面都能找到 这里就不写了

在Spring配置文件中添加

    <bean id="Account_Set_P" class="com.htsa.www.Bens.entity.Account_Set" p:name="hu" p:age="12" p:money="22.22"/>
    <bean id="Account_Ref_P" class="com.htsa.www.Bens.entity.Account_Ref" p:id="1" p:set-ref="Account_Set_P"/>
@Test
public void show8() {
    ApplicationContext context = new ClassPathXmlApplicationContext("ApplicationContext.xml");
    //使用set的方式
    Account_Ref account_Ref_set = (Account_Ref) context.getBean("Account_Ref_P");
    System.out.println(account_Ref_set);
}

Account_Ref{id=1, set=Account_structure{name=’hu’, age=12, money=22.22}}

使用注解实现IOC

不写 默认为singleton(单列模式)

@Scope("prototype")//设置为多例 写在类的上面

类似于这样

@Component("Account_comment")
@Scope("prototype")
public class Account_comment implements IAccountDao {

以下4种注解类 都是用于对象装载的 ,效果一样 主要就是更加语义化而已,写在类的上面

       @Component :组件Bean层   javaBean(实体类)

       @Repository : 持久化DAO层  Mybatis     

       @Service : 业务逻辑Service层  

       @Controller : 表现层    Servlet

      注意名称唯一

以下三种注解是用来将类 装配 到属性里 需要用到注解的名称 也就是上面四个注解声明的名称或者配置文件中的 bean的 id 都行

可以将类装配到接口 也可以类装配到类

@Autowired注解装配类属性 自动注入

@Qualifier 配合@Autowired来使用 装配指定 Bean

@Resource 注解 装配 等同于上面 两个注解的结合 (常用)

使用@Value(“?”) 来对基本数据类型 和字符串进行赋值

添加注解扫描仪 用于解析指定包下的注解 多个包之间可以用 逗号隔开

<context:component-scan base-package="com.htsa.www.Bens.entity"/>


<context:component-scan base-package="com.htsa.www.Bens.entity,com.htsa.www.Bens.entity"/>

Spring自动装配

通过配置文件中的bean 来自动装配 指定bean中的对象属性

在实际开发中 是不推荐使用自动装配的 开发的时候你可能会知道谁是谁 但是时间长了 你根本不知道 谁给谁赋值了 (依赖关系不明确)

还容易引发潜在的错误 ,造成后期 维护困难 难以阅读代码

但是如果合理的 进行 自动装配的设计 还是可以提高 代码的 质量的

比如单独一个配置 文件中 全都是 byName装配 或者 单独一个配置文件中全都是byType 将不能进行装配的 bean给提取出来单独处理 比如数据访问的一系列操作 和 Mybatis的一系列操作 和AOP切面 这些不是自己写的类 都不要进行自动装配

(要提前设计好)

属性详情
no默认值. Spring 默认不进行转配,必须显示指定依赖对象 也就是手动进行装配
byName根据属性名自动装配.Spring 自动查找于属性名相同的id ,如果找到了,则自动注入 ,否则什么都不做
byType根据属性的类型自动装配,Spring自动查找于属性类型相同的Bean,如果刚好找到唯一的那个,则自动注入 如果找到多个 与属性类型相同的Bean ,则抛出异常 ,如果没找到,就什么也不做
constructor和byType类似 ,不过他针对构造方法 ,如果Spring找到一个Bean和构造方法的参数类型相匹配.则通过构造注入 该依赖对象 ,如果找不到,将抛出异常

byName 通过 bean id 名称 来进行注入 因为 id 的名称都是 不能重复的 所以 错误率比较低

使用byType 呢 就有可能发生 多个bean类型相同的 注入 异常 他是按照class类型来进行装配;的

而constructor 呢 一般都是不会使用的 我赋值还需要单独在写 一个构造啊 没事干了吧

byName用法

<bean id="userMapper" class="com.bdqn.www.dao.UserMapperimpl"></bean>

<bean id="UserServiceimpl" class="com.bdqn.www.service.UserServiceimpl" autowire="byName"> </bean>

UserServiceimpl 类中有 一个 userMapper的属性
private UserMapper userMapper;

autowire="byName"  是按名字 进行匹配的   <bean id="userMapper"  符合  就会自动进行 注入

byType用法

<bean  class="com.bdqn.www.dao.UserMapperimpl"></bean>

<bean id="UserServiceimpl" class="com.bdqn.www.service.UserServiceimpl" autowire="byType"> </bean>


UserServiceimpl 中有 一个 userMapper的接口属性 
private UserMapper userMapper;

bean UserMapperimpl类实现了UserMapper的接口  
所以 第一个bean  符合UserServiceimpl 中userMapper这个属性类型 会自动进行 注入

这样的情况 就会报错
<bean   class="com.bdqn.www.dao.UserMapperimpl_2"></bean>

<bean   class="com.bdqn.www.dao.UserMapperimpl"></bean>

<bean id="UserServiceimpl" class="com.bdqn.www.service.UserServiceimpl" autowire="byType"> </bean>

这两个bean  都 继承了 UserMapper  所以 是类型相同    这时候 
autowire="byType"  就不知道 到底使用哪一个bean  自动进行 注入 了

constructor用法

<bean   class="com.htsa.www.Bens.entity.UserMapperimpl"/>

<bean id="UserServiceimpl" class="com.htsa.www.Bens.entity.UserServiceimpl" autowire="constructor"/>

xml配置文件 bena自动装配

定义default-autowire 后 此文件下所有的bean 都开启了指定的装配模式 不需要在每个bean标签上写了

    <beans xmlns="http://www.springframework.org/schema/beans"
           xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
           xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd"
           default-autowire="byName">

在beans标签设置default-autowire=”byName”后，在spring容器实例化bean（包含xml配置bean和注解方式bean）时，该bean内部的字段会自动根据byName规则 自动装载spring容器里的bean

还可以使用byType constructor

ApplicationContext context=new ClassPathXmlApplicationContext(“ApplicationContext.xml”);
UserDaoImpl im=(UserDaoImpl)context.getBean(“UserDaoImpl”);
