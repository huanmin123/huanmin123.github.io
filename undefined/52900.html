java_Mybatis_Mapper接口文件解析

官方教学

动态绑定(普通)

<mapper namespace=”cn.htsa.www.dao.UserMapper”>

来关联mapper接口 通过UserMapper mapper =sqlSession.getMapper(UserMapper.class);来使用接口内的 方法

接口内的方法名是个mapper.xml文件中的 id名关联的(一致)

简单的演示一下:

Mapper.xml 文件

<!--命名空间 绑定 接口类-->
<mapper namespace="cn.htsa.www.dao.UserMapper">

<!--查询全部数据条数-->
    <select id="count" resultType="int" >
           select count(*) from product
    </select>

    .....
</mapper>

Mapper.java (接口)

public interface UserMapper {
    public  int  count();
}

在接口的参数中可以指定@Param(“”) 来设置 参数的别名

public books getUserListBybooksID(@Param(“id”) int num,@Param(“pass”) String password);

在 mapper.xml 获取这个接口的参数 #{id} 和#{pass}

获取数据库数据

 SqlSession sqlSession = MyBatisUtil.createSqlSession();
//通过接口的方式来 访问数据库
UserMapper mapper = sqlSession.getMapper(UserMapper.class);
        int count = mapper.count();
        System.out.println(count);

//关闭会话 释放资源
  MyBatisUtil.closeSqlSession(sqlSession);

使用注解的方式(进阶)

使用注解的方式就是脱离mapper.xml文件 直接在接口方法上写sql语句

这样好处就是方便,不好之处就是 拼接语句比较费劲 容易错误 ,一般简单的sql语句使用注解比较好,如果太复杂的话最好还是使用其他的方式,

注意事项如果使用了注解 那么此包下的Mapper.xml就不能和接口Mapper重名了 否则就报错了

常用的注解

@Select 查询
@Insert 添加
@Update 修改
@Delete 删除
特别复杂的 sql语句使用动态sql类进行拼接

@Results 映射 必须使用在方法上才有效 单独写,程序是不会读取的 别人的方法也无法通过id调用了 注意 实体类中属性和数据库中字段不同的可以使用 如果相同的可以省略,当然如果出现查询空字段那么你还是要指定的
@Result 映射配置

@One 一对一
@Many 一对多

​

@ResultMap 调用映射 (调用以使用过的映射)

第一步先 将mapper位置所在包添加到Mybatis核心配置文件中

    <mappers>
     <package name="com.bdqn.www.dao"/>
    </mappers>

有些人是使用 class 的 这样多个mapper就会报错的 所以还是使用package最好

这样还可以和Mapper.xml同时使用 只要将namespace命名空间指向对应的Mapper接口就行了

第二步就是在dao层下创建一个接口就行了

在添加里还有一个注解是@SelectKey用来获取添加后的元素在数据库中字段的id值

  @Insert("insert into product values(default,#{name},#{price},#{type_id})")
  @SelectKey(keyProperty = "id",keyColumn = "id" ,before = false,resultType = Integer.class,statement = {"select last_insert_id()"})
    public  int insert_in(product product);

keyProperty是实体类中的字段

keyColumn是数据库中的id主键

before = false 在之后 resultType返回类型

statement 执行语句 固定写法

动态sql

当然动态sql也是能使用的 只要你不嫌麻烦随意(可以参考 Mapper.xml文件)

使用动态sql必须使用<script> </script> 来包围语句 否则不解析一直报错

但是在无法使用大于号 、小于号，那如何解决这问题呢，其实只要把大于号、小于号转义即可&nbsp;(大于号) &lt;(小于号)

这里就演示一个

    @Select("<script>"+ "select  name,type_id from product "+
            "<trim prefix=\"where\" prefixOverrides=\"and|or\">"+
            "<if test=\"name !=null and name !=''\">"+
            " name = #{name} "+
            "</if>"+
            "<if test=\"price !=null and price !=''\">"+
            "or price = #{price} "+
            "</if>"+
            "</trim>"
        +"</script>"
    )
    public List<product> show2(@Param("name") String name,@Param("price") Double price);

所以为劝你 别瞎搞 太费劲了

通过以上案例 发现没 使用注解 在复杂的sql语句上还不如 使用xml文件来的方便

而且搞不好一个空格 就不能用了 或者出现各种错误 看都看不出来 为了解决以上问题

使用注解的方式[高阶]

这种方式 可以使用java代码 你平时JDBC怎么玩字符串拼接 这里面就怎么玩 if 循环 switch 等等 都能使用 特别方便 实现动态sql不在是梦想 也没那么费劲了同时你注解该怎么用还怎么用不冲突的 也就是说 简单的sql 直接使注解来实现 困难的sql 就在自定义sql类里实现 ,这样就不用担心在写错代码了 ,困难的sql都是使用java原生代码拼接的 错了很好找错误 , 不像使用注解拼接费劲

XML配置方式的动态SQL我就不讲了上面有，有兴趣可以自己了解，下面是用<script>的方式把它照搬过来，用注解来实现。适用于xml配置转换到注解配置

@Select("<script>select * from user <if test=\"id !=null \">where id = #{id} </if></script>")  

public List<User> findUserById(User user);

很明显，在java中写xml可读性和维护性太差，尤其当SQL很长时，这样写是很痛苦的。在这里就不多说了。

操作方法不变还是使用MyBatisUtil工具类里来调用(可以参考注解的调用)

*特别注意事项 *

不管是拼接sql语句 还是使用注解声明 类和方法 都不能有空格否则报错
不要使用重载，也就是说，不要使用方法名相同参数不同的方法，以避免发生诡异问题。
接口方法的 参数必须使用@Param(“”) 进行注解 否则 无法形成Map 就会报错 实体类不用
在接口中以实体类作为参数的时候 , 只能是唯一参数 不能在有其他的参数了

在sql类中所有方法都是返回字符串类型的,

方法参数(map,和实体类,或者无参就这三种),

方法名可以可以和接口的方法名相同(这样省的在去想名称了)

实体类映射还是需要使用@Results 来映射 以及一对一,和一对多

在sql类中拼接sql的二种方法

  1. 字符串拼接 这个我就不用说了 傻子都会的天天玩JDBC都练吐了.

  2. 使用sql对象 这个对象里封装了基本的sql语句

获取参数的方法

第一种直接使用#{} 直接获取方法中的参数 或者 实体类中属性 获取对应的值了(名称要相同)
第二种使用Map获取方法参数 这种方法 必须指定@Param(“”)
第三种 使用实体类的方式 这种方式 就不用说了 很简单
第四种 使用 Map获取 获取 传进来的Map参数

第五种使用单引号拼接"'"+内容+"'" 因为数据库只支持单引号

第六种使用${} 这个和#{}区别是 ${}可以参与字符串拼接 而#{}不行

用为#{}的值 是 ? 只有最后 要将内容传给数据库的时候 ?才会变成实际值

我只到你在想什么 是无法使用array 和list set 这些集合或者数组直接作为方法的参数的 ,为什么map可以 因为map 是key/value 而参数也是 key/value所以 他们其实都是一个样子的 你可以变通下 -将数组等 ..存储在map中 利用key在获取的方式
