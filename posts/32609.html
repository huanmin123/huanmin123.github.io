java-MyBatisUtil工具类文件解析

官方教学


import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import java.io.IOException;
import java.io.InputStream;
public class MyBatisUtil {
    private static SqlSessionFactory factory;

    static {
        try {
            String resource = "mybatis-config.xml";
            //将MyBatis核心配置文件加载到输入流中
            InputStream is = Resources.getResourceAsStream(resource);
            //然后创建一个会话工厂
            factory = new SqlSessionFactoryBuilder().build(is);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    //创建SqlSession  并 打开事务
    public static SqlSession createSqlSession() {
        return factory.openSession(false);
    }

    //增删改查中 都要使用     sqlSession.commit();//提交事务   sqlSession.rollback();回归事务  最后调用下方法关闭会话

    //关闭SqlSession 会话
    public static void closeSqlSession(SqlSession sqlSession){
        if(sqlSession!=null){
            sqlSession.close();
        }
    }
}

SqlsessionFactoryBuilder

用过即丢,其生命周期只存放于方法体

可重用其来创建多个 SqlSessionFactory实例

负责构建SqlSessionFactory ,并提供多个build方法的重载

build(inputStream inputStream ,String environment ,Properties properties)

build(Reader reader,String environment,Properties properties)

build(Configuration config)

配置信息以三种形式提供给SqlSessionFactory的build方法

inputStream(字节流) , Reader (字符流) , Configuration(类)

我们使用的 是 字节流(传入的是配置文件xml)

InputStream is = Resources.getResourceAsStream(resource);

SqlSessionFactory

是每个MyBatis应用的核心 ,作用:创建SqlSession实例

SqlSession session=sqlSessionFactory.openSession(boolean autoCommit);

默认为true 关闭事务控制 而false 是打开事务控制

sqlSession.commit();//提交事务
sqlSession.rollback();//回滚事务

作用域:Application

生命周期与应用的生命周期相同

单列(在整个程序之间只有一个实例)

存在于整个应用运行时,并且同时只存在一个对象实例

获取SqlSessionFactory的代码是否可用进行优化

静态代码块 ,以保证SqlSessionFactory只被创建一次

SqlSession

包含了执行sql所需的所有方法

对应一次数据库会话,会话结束必须关闭

线程级别,不能共享

作用域 在一次 线程 或者一次 方法中 也就是一个request请求之间

关闭SqlSession非常重要 必须确保在Finally代码块中正常关闭

在SqlSession里可以执行多次sql 语句 ,但一旦关闭了( close)

SqlSession就需要重新 创建

通过SqlSession实例运行sql语句

int count=sqlSession.selectOne(“com.bdqn.dao.UserMapper.count”);

com.bdqn.dao.UserMapper 是xml文件所在位置 而后面的count 是 Mapper元素的 id名

但是这种方式 缺点就是 如果 字符串写错了 就找不到了

SqlSession 中执行sql语 常用的 几种

selectOne 查询一条显示的(selest ? from where ….) 信息

selectList 查询多条显示的 (select * from …) 信息 以List的方式 适用于 将数据库中的多条数据 提取出来 放入List集合

insert 添加 返回 执行结果条数

delete 删除 返回执行结果 条数

基于Mapper接口方式操作数据

使用接口实现调取Mapper中的sql语句 接口中的方法名必须和Mapper中的id一致

而且在Mapper.xml中必须配置接口或者

<mapper namespace=”com.bdqn.dao.UserMapper”>…</mapper>

List<User> userList =sqlSession.getMapper(UserMapper.class).getUserList();

注意事项: UserMapper.class就是引用此接口 而后面的getUserList() 是接口内的方法 , 会根据接定义的方法 来执行sql语句

*总结 *

创建SqlSessionfactoryBuilder的作用其实就是 为了创建 SqlSessionFactory 而创建SqlSessionFactory 就是为了创建SqlSession

   @Test
    public void show() {

        SqlSession sqlSession = MyBatisUtil.createSqlSession();
        //通过接口的方式来 访问数据库
        UserMapper mapper = sqlSession.getMapper(UserMapper.class);

        } catch (Exception e) {
