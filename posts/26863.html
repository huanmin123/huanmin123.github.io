原型模式

介绍

原型模式（Prototype Pattern）是用于创建重复的对象，同时又能保证性能。这种类型的设计模式属于创建型模式，它提供了一种创建对象的最佳方式。

这种模式是实现了一个原型接口，该接口用于创建当前对象的克隆。当直接创建对象的代价比较大时，则采用这种模式。例如，一个对象需要在一个高代价的数据库操作之后被创建。我们可以缓存该对象，在下一个请求时返回它的克隆，在需要的时候更新数据库，以此来减少数据库调用。

优点： 1、性能提高。 2、逃避构造函数的约束。

缺点： 1、配备克隆方法需要对类的功能进行通盘考虑，这对于全新的类不是很难，但对于已有的类不一定很容易，特别当一个类引用不支持串行化的间接对象，或者引用含有循环结构的时候。 2、必须实现 Cloneable 接口。

使用场景： 1、资源优化场景。 2、类初始化需要消化非常多的资源，这个资源包括数据、硬件资源等。 3、性能和安全要求的场景。 4、通过 new 产生一个对象需要非常繁琐的数据准备或访问权限，则可以使用原型模式。 5、一个对象多个修改者的场景。 6、一个对象需要提供给其他对象访问，而且各个调用者可能都需要修改其值时，可以考虑使用原型模式拷贝多个对象供调用者使用。 7、在实际项目中，原型模式很少单独出现，一般是和工厂方法模式一起出现，通过 clone 的方法创建一个对象，然后由工厂方法提供给调用者。原型模式已经与 Java 融为浑然一体，大家可以随手拿来使用。

注意事项：与通过对一个类进行new实例化来构造新对象不同的是，原型模式是通过拷贝一个现有对象生成新对象的 注意是新的对象也就是地址也变了 但是拷贝的内容 分为深拷贝和浅拷贝 那么什么是浅拷贝 什么是深度拷贝

深拷贝和浅拷贝最根本的区别在于 如果类中存在引用类型变量 (字符串和包装类型除外)

深度拷贝 会将对象中的变量包括引用类型 都进行拷贝一份新的 到内存中

浅度拷贝 就只会拷贝基本数据类型 , 包装类型和字符串的值 到内存中 不会拷贝引用类型

假设B复制了A，修改A的引用类型变量时候，看B是否发生变化：

如果B的引用类型变量 跟着也变了，说明是浅拷贝，

如果B的 引用类型变量 没有改变，说明是深拷贝，

下面会有 教你测试是浅拷贝和深度拷贝

浅拷贝实现 Cloneable，重写clone，深拷贝是都继承 Serializable 然后重写clone 读取二进制流。

看图感觉和工厂模式差不多 但是 注意 我们是通过Map集合的方式 先将所有需要的对象都存入进去 作为样本

然后需要的时候通过样本 给你一个新的对象

在正常创建一个对象会 经过很多到程序

1：检查类是否已经被加载；

2：为对象分配内存空间；

3：为对象字段设置初始值；

4：设置对象头；

5：执行构造方法。

我们使用拷贝直接省略这些步骤 原理就是将样本对象 复制一份放入新的内存空间

这样就能提升 创建对象的效率了

下面案例会将浅拷贝和深度拷贝都实现出来

浅拷贝

创建一个Shape(形状抽象类) 并继承 Cloneable接口 重写clone

public abstract class Shape implements Cloneable {

    private String id;  //形状id
    private String type; //形状
    public abstract void draw(); 

    public String getId() {
        return id;
    }

    public void setId(String id) {
        this.id = id;
    }

    public String getType() {
        return type;
    }

    public void setType(String type) {
        this.type = type;
    }

    @Override
    public Object clone() {
        Object clone = null;
        try {
            clone = super.clone();
        } catch (CloneNotSupportedException e) {
            e.printStackTrace();
        }
        return clone;
    }
}

public class ShapeCache {

    private static Hashtable<String, Shape> shapeMap = new Hashtable<String, Shape>();

    // 当前类加载时候 将所有原型都存入 Hashtable中
   static {
        Circle circle = new Circle();
        circle.setId("1");
        shapeMap.put(circle.getId(),circle);

        Square square = new Square();
        square.setId("2");
        shapeMap.put(square.getId(),square);

        Rectangle rectangle = new Rectangle();
        rectangle.setId("3");
        shapeMap.put(rectangle.getId(),rectangle);
    }

    //返回一个拷贝的副本的新对象 
    public static Shape getShape(String shapeId) {
            Shape cachedShape = shapeMap.get(shapeId);
            return (Shape) cachedShape.clone();
    }

}

深度拷贝

网上有很多深度拷贝的方法 但是都不推荐 只建议使用下面这一种方式

串行化深拷贝

必须所有Shape抽象类的实现类都必须实现Serializable接口 也就是 在抽象类上添加Serializable接口 那么他的子类也就自动继承了

public abstract class Shape implements Cloneable,Serializable{
xxx
}

然后修改Shape 抽象类的clone方法 将原来的浅拷贝的clone() 删除 换成下面的

    //深度拷贝
    @Override
    public Object clone() {
        Object object = null;
        try {
            // 序列化
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);

            oos.writeObject(this);

            // 反序列化
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            object = ois.readObject();
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
        return object;
    }

然后将 Objtest 这个类和他的内部类Objtest1都 继承Serializable 接口 否则无法被 序列化 就会报 NotSerializableException xxx

浅拷贝和深度拷贝结合版

很简单 弄2个clone() 不就行了

在上面的案例中 然后修改Shape 抽象类 代码如下

public abstract class Shape implements Cloneable, Serializable {

    private String id;  //形状id
    private String type; //形状
    public Objtest obj=new Objtest();
    public abstract void draw();

    public String getId() {
        return id;
    }

    public void setId(String id) {
        this.id = id;
    }

    public String getType() {
        return type;
    }

    public void setType(String type) {
        this.type = type;
    }



    //浅度拷贝
    @Override
    public Object clone()  {
        Object object = null;
        try {
            object= super.clone();
        } catch (CloneNotSupportedException e) {
            e.printStackTrace();
        }
        return object;
    }

    //深度拷贝
    public Object  deepClone() { //自定义的Clone
        Object object = null;
        try {
            // 序列化
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);

            oos.writeObject(this);

            // 反序列化
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            object = ois.readObject();
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
        return object;
    }
}

然后我们在原型工厂 ShapeCache 里修改下代码如下:

public class ShapeCache implements Serializable {

    private static Hashtable<String, Object> shapeMap = new Hashtable();

    // 当前类加载时候 将所有原型都存入 Hashtable中
    static {
        Circle circle = new Circle();
        circle.setId("1");
        shapeMap.put(circle.getId(),circle);

        Square square = new Square();
        square.setId("2");
        shapeMap.put(square.getId(),square);

        Rectangle rectangle = new Rectangle();
        rectangle.setId("3");
        shapeMap.put(rectangle.getId(),rectangle);
    }


    /**
     *
     * @param shapeId
     * @param pd   如果为true就是 深度拷贝 如果是false 那么就是浅拷贝
     * @return Shape
     */
    public static Shape getShape(Object shapeId,boolean pd) {

        if (pd){
            //深度拷贝
            Shape cachedShape =(Shape) shapeMap.get(shapeId);
            return (Shape) cachedShape.deepClone();

        }else{
            //浅拷贝
            Shape cachedShape =(Shape)shapeMap.get(shapeId);
            return (Shape) cachedShape.clone();
        }

    }
    //重载浅拷贝
    public static Shape getShape(Object shapeId) {
        Shape cachedShape =(Shape) shapeMap.get(shapeId);
        return (Shape) cachedShape.clone();
    }


}

为甚会这样 因为 不论是深度拷贝还是浅度拷贝 都是给你一个原型对象的副本 既然是副本那么就好比是复制一个文件一样 内容都给你复制过去了

唯一区别就是如果原型对象中出现 对象属性的话(字符串和包装类除外) 并且有值

那么浅拷贝给你的是一个地址 而深度拷贝给你的是全新的对象属性(也就是相当于又new了一次)
